# Obsidian Relay Sync

A plugin for Obsidian that provides reliable and secure synchronization between devices through a relay server.

[По-русски](#русская-версия)

## Features

- 🔐 **Security**: End-to-end encryption, the server has no access to content
- 🔄 **Reliability**: Works through a relay server, bypassing P2P connection limitations
- 📱 **Cross-platform**: Full functionality on desktop and mobile devices
- 🗝️ **Trusted device system**: Key management and sync confirmations
- ⏱️ **Flexible settings**: Temporary keys, automatic disconnect, trusted device management

## Installation

### From Obsidian Community Plugins

1. Open Obsidian Settings
2. Go to Community Plugins
3. Turn off Restricted Mode
4. Click "Browse" and search for "Relay Sync"
5. Install the plugin and enable it

### Manual installation

1. Download the latest release
2. Extract the files to your Obsidian plugins folder: `{vault}/.obsidian/plugins/obsidian-relay-sync`
3. Restart Obsidian
4. Enable the plugin in Settings → Community Plugins

## Usage

1. Open plugin settings and enter your relay server URL
2. Set an encryption password (same on all devices)
3. Click "Connect" to start syncing
4. On other devices, use the invitation key system to establish trusted connections

# План оптимизации синхронизации Obsidian Relay Sync

## Базовая стратегия

✅ **Стратегия инициализации:** При запуске плагин выполняет одноразовое сканирование всех файлов и создает индекс с метаданными (пути, хеши, время модификации). Затем сравнивает этот индекс с индексами других устройств, чтобы определить необходимость синхронизации.

✅ **Периодическая легкая проверка:** Вместо постоянного наблюдения за всеми файлами, плагин периодически (каждые 5 минут) проверяет только метаданные файлов и сравнивает с сохраненным индексом.

✅ **Выборочный мониторинг:** Отслеживание только активно редактируемых файлов. Файл добавляется в список активного мониторинга при открытии для редактирования, а после закрытия через некоторое время удаляется из этого списка.

✅ **Сигнальная система:** Когда на одном устройстве обнаружено изменение, отправляется только сигнал другим устройствам с метаданными файла (путь, хеш, размер). Другие устройства сначала проверяют, нужна ли им новая версия, и только при необходимости запрашивают содержимое.

✅ **Умная очередь синхронизации:** Приоритизация синхронизации недавно измененных файлов и тех, которые открыты на других устройствах.

⚠️ **Настраиваемые интервалы:** Пользователь должен иметь возможность настраивать интервал проверки в зависимости от своих потребностей и производительности устройства.

⚠️ **Режим гибернации:** Если устройство бездействует определенное время, переводить плагин в "спящий режим" с минимальной активностью и проверками.

## Дополнительные оптимизации

✅ **Кэширование:** Интеллектуальное кэширование для часто изменяемых файлов - реализовано через `fileContentCache`.

✅ **Дельта-синхронизация:** Для Markdown файлов передаются только измененные части - реализовано с использованием алгоритма сравнения по общим началу и концу.

✅ **Сжатие данных:** Для текстовых файлов реализовано сжатие через словарь повторяющихся строк.

⚠️ **Обработка конфликтов:** Отложено на будущее, так как система используется одним пользователем.

⚠️ **Визуальная обратная связь:** Отложено на будущее, планируется реализация лога синхронизации.
